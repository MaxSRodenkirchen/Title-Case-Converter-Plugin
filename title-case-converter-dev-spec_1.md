# Obsidian Plugin: "Title Case Converter" — Developer Spec

Give this entire document to a coding AI (e.g. Claude Code) as the build spec.

## 1. Goal

Build an Obsidian plugin that converts the currently **selected text** in the
editor into proper title case, following a specific citation style's
capitalization rules. Version 1 ships with **AMA style** only, but the
architecture must make it trivial to add more styles (APA, Chicago, MLA, ...)
later as additional commands, without touching the core logic.

## 2. Tech constraints

- TypeScript, standard Obsidian plugin scaffold (`manifest.json`, `main.ts`,
  `esbuild.config.mjs`, as generated by the official
  [obsidian-sample-plugin](https://github.com/obsidianmd/obsidian-sample-plugin)).
- Use only **current, non-deprecated** Obsidian API methods. Specifically:
  `this.addCommand()` with `editorCallback: (editor: Editor) => void`,
  `editor.getSelection()`, `editor.replaceSelection()`. Do not use any API
  flagged deprecated in the current Obsidian API docs.
- No external runtime dependencies beyond what's needed (this should be
  doable with zero npm dependencies besides Obsidian's own types).
- Free/open-source only — no paid API calls, no telemetry.
- If nothing is selected, show a `Notice` ("Please select text first") and
  do nothing.

## 3. Architecture

### 3.1 Style module interface

Each capitalization style is a self-contained module implementing:

```ts
interface TitleCaseStyle {
  id: string;          // "ama", "apa", ...
  label: string;        // "AMA", "APA", ... (used in command name & settings)
  convert(input: string, exceptions: string[]): string;
}
```

All styles live in `src/styles/`, one file per style (e.g. `ama.ts`). A
central registry (`src/styles/index.ts`) exports an array of all available
styles. `main.ts` loops over this array and registers one command per style:

```
"Convert selection to Title Case (AMA)"
```

Adding a new style later = add one file + one array entry. No changes to
`main.ts` needed.

### 3.2 Settings

A settings tab with:

- **Exception list**: a multi-line textarea, one word/phrase per line, for
  words whose casing should always be preserved exactly as the user typed
  it (e.g. brand names, personal exceptions). Stored as `string[]` in plugin
  data.
- **Auto-preserve mixed-case words** (toggle, default ON): if a word already
  contains an uppercase letter *after* its first character (e.g. `NASA`,
  `iPhone`, `macOS`, `GitHub`), leave that word completely untouched,
  regardless of AMA rules. This works automatically without needing the word
  in the exception list.

Both mechanisms combine: a word is left untouched if it matches the
exception list **OR** the mixed-case heuristic (when enabled).

## 4. AMA title case rules (for `src/styles/ama.ts`)

Implement these rules exactly:

1. **Always capitalize** the first and last word of the (sub)title,
   regardless of word length or type.
2. **Capitalize major words**: nouns, pronouns, verbs, adjectives, adverbs,
   subordinating conjunctions — in practice: capitalize every word **except**
   the ones matched by rule 3.
3. **Do not capitalize** (lowercase) the following, unless they are the
   first or last word:
   - Articles: `a`, `an`, `the`
   - Coordinating conjunctions: `and`, `but`, `for`, `nor`, `or`, `so`, `yet`
   - Prepositions of three or fewer letters: `as`, `at`, `by`, `for`, `in`,
     `of`, `off`, `on`, `out`, `per`, `to`, `up`, `via`
   - The word `to` when used in an infinitive (**simplification**: since
     reliable infinitive detection requires real grammar parsing, always
     lowercase `to` unless it's the first/last word — this matches how
     titlecaseconverter.com and similar tools handle it in practice).
4. **Hyphenated compounds**: capitalize both parts independently, each
   following rules 1–3 as if it were its own word within the title (e.g.
   `well-known` → `Well-Known`, `x-ray` → `X-Ray`, but `check-in` at a
   non-edge position stays `Check-In` since both parts are major words —
   only lowercase a hyphen-part if it individually matches rule 3 and is not
   the very first/last word of the whole title).
5. Titles containing a colon (title + subtitle) are treated as **two title
   segments**: capitalize the first word of each segment (i.e. first word
   after the colon is always capitalized too), and the last word of each
   segment follows rule 1 as well.
6. Genus/species Latin binomials (e.g. *Escherichia coli*) — **out of scope
   for v1**, skip this rule. Leave as a `// TODO: genus/species capitalization
   not implemented` comment in the code so it's easy to find later.

### 4.1 Word list source (put in `src/styles/ama.ts` as constants)

```ts
const ARTICLES = ["a", "an", "the"];
const COORDINATING_CONJUNCTIONS = ["and", "but", "for", "nor", "or", "so", "yet"];
const SHORT_PREPOSITIONS = ["as", "at", "by", "for", "in", "of", "off", "on", "out", "per", "to", "up", "via"];
const LOWERCASE_WORDS = new Set([...ARTICLES, ...COORDINATING_CONJUNCTIONS, ...SHORT_PREPOSITIONS]);
```

### 4.2 Algorithm outline

```
function convert(input, exceptions):
  segments = split input on ":" (keep the colon with the segment before it)
  for each segment:
    words = split segment on whitespace (keep original whitespace/punctuation
            attached where sensible — see 4.3 on tokenization)
    for i, word in words:
      isEdge = (i == 0) or (i == words.length - 1)
      if word matches exception list OR (mixed-case heuristic enabled AND
         word has an uppercase letter after position 0):
        leave word untouched
        continue
      if word contains a hyphen:
        apply this same per-word logic to each hyphen-part independently
        continue
      lowerWord = word.toLowerCase()
      if isEdge or lowerWord not in LOWERCASE_WORDS:
        capitalize first letter of word, rest lowercase
      else:
        word = lowerWord
  rejoin segments and words, preserving original whitespace
```

### 4.3 Tokenization detail

- Split on whitespace, but keep leading/trailing punctuation attached to the
  word for the capitalization check (strip it temporarily, capitalize the
  letter, reattach). E.g. `"idle"` → `"Idle"`, `(idle)` → `(Idle)`.
- Preserve the original whitespace between words exactly (don't collapse
  multiple spaces) so the command is safe to use on already-formatted text.
- First/last word detection (rule 1) must look at the first/last
  **alphabetic token**, ignoring leading/trailing punctuation-only tokens.

## 5. Testing — required test cases

Write unit tests (plain TS, no framework needed, or Vitest if you prefer)
covering at least:

| Input | Expected AMA output |
|---|---|
| `how to be idle` | `How to Be Idle` |
| `a tale of two cities` | `A Tale of Two Cities` |
| `well-known facts about nasa` | `Well-Known Facts About NASA` *(if "nasa" typed lowercase, mixed-case heuristic does NOT apply — only real existing mixed case is preserved; this case should actually capitalize normally to `Well-Known Facts About Nasa` unless "NASA" is in the exception list)* |
| `the lord of the rings: the fellowship of the ring` | `The Lord of the Rings: The Fellowship of the Ring` |
| `up and out: a memoir` | `Up and Out: A Memoir` |
| `check-in procedures` | `Check-In Procedures` |
| `working with iPhone apps` | `Working With iPhone Apps` *(iPhone preserved via mixed-case heuristic)* |

Add a couple more edge cases yourselves (single word, empty string, string
with only lowercase small words, leading/trailing whitespace).

## 6. UX details

- Command name in the command palette: `Convert selection to Title Case (AMA)`.
- If no selection: `Notice` "Title Case Converter: please select some text first."
- After conversion, cursor/selection state should remain sane (Obsidian's
  `replaceSelection` handles this by default — just verify it feels right).
- Plugin ID: `title-case-converter`. Plugin name: `Title Case Converter`.

## 7. Distribution: Obsidian Community Plugins

The plugin is intended to eventually be submitted to Obsidian's official
Community Plugins directory, so build it to that standard from day one.

### 7.1 Repo structure requirements

- Public GitHub repo, one plugin per repo.
- `manifest.json` at repo root with exactly these fields:
  ```json
  {
    "id": "title-case-converter",
    "name": "Title Case Converter",
    "version": "1.0.0",
    "minAppVersion": "1.4.0",
    "description": "Convert selected text to proper title case using citation-style rules (AMA, with more styles planned).",
    "author": "<your name>",
    "authorUrl": "<optional>",
    "isDesktopOnly": false
  }
  ```
  - `id` must be unique, lowercase, hyphenated, and must not contain the
    word "obsidian".
  - `name` also must not contain "Obsidian" (branding guideline).
  - Description should be plain, factual, no marketing language (per
    Obsidian's plugin guidelines).
- `versions.json` at repo root, mapping plugin version → minimum required
  Obsidian app version, e.g.:
  ```json
  { "1.0.0": "1.4.0" }
  ```
- `README.md` (usage, screenshots/gif of the command in action, how to add
  a style module for contributors) and a `LICENSE` file (MIT is the common
  choice for Obsidian plugins).
- `.gitignore` should exclude `node_modules`, `main.js` (build output does
  **not** need to be committed to the repo — it's shipped via Releases, see
  below), and `.DS_Store`.

### 7.2 Release workflow

Obsidian loads community plugins by downloading `main.js`, `manifest.json`
(and `styles.css` if present) from the **assets of a GitHub Release**, not
from the repo source directly. So:

- Every version bump needs a GitHub Release whose **tag name matches the
  version number exactly** (e.g. tag `1.0.0`, not `v1.0.0`), with `main.js`,
  `manifest.json`, and `styles.css` (if used) attached as binary release
  assets.
- Set up a small GitHub Actions workflow (`.github/workflows/release.yml`)
  that triggers on tag push, runs `npm run build`, and uploads the three
  files to the release automatically — this avoids manual mistakes. Ask the
  coding AI to include this workflow file as part of the deliverables.

### 7.3 Submission to the directory

- One-time step, done manually by you (not the coding AI): fork
  `obsidianmd/obsidian-releases`, add an entry for the plugin to
  `community-plugins.json`, open a PR. Obsidian's bot + a human reviewer
  check guideline compliance (no `eval()`, no obfuscated code, no
  unnecessary network requests, correct API usage, etc.).
- Review turnaround varies (days to a few weeks). Once merged, the plugin
  becomes installable via Obsidian's in-app Community Plugins browser, and
  future releases (new tags) are picked up automatically without a new PR.
- Until the PR is merged, you can already use and test the plugin locally
  by copying `main.js`, `manifest.json` (and `styles.css`) into
  `<vault>/.obsidian/plugins/title-case-converter/` — mention this as the
  dev/testing install method in the README too.
